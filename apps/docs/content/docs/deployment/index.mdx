## Overview

Production is deployed automatically via **GitHub Actions** when code is pushed to `main`. The stack runs in Docker containers behind a **Traefik** reverse proxy with Let's Encrypt SSL.

## Deployment Options

### Automatic (recommended)

Push to `main` triggers the deployment workflow automatically.

### Manual via GitHub Actions

Go to **Actions** tab and run "Deploy to Production" with options:

| Option | Values | Description |
|--------|--------|-------------|
| Deployment type | `full`, `quick`, `service-only` | Full rebuild, restart only, or specific services |
| Services | `api`, `web`, `redis`, `traefik` | Which services to deploy |
| Skip build | `true`/`false` | Deploy without rebuilding images |

### Emergency

```bash
./deploy-manual.sh              # Full deploy
./deploy-manual.sh --quick      # Restart without rebuild
./deploy-manual.sh --service api,web
```

## Rollback

Run the "Rollback Deployment" workflow in GitHub Actions. Options:
- Rollback to previous deployment
- Specify a commit hash

## Health Checks

Run the "Health Check" workflow to verify all services:
- Redis connectivity
- API responsiveness
- Web accessibility
- Traefik routing
- SSL certificate validity

## Required GitHub Secrets

| Secret | Description |
|--------|-------------|
| `SSH_PRIVATE_KEY` | SSH key for server access |
| `DEPLOY_SERVER` | Production server hostname/IP |
