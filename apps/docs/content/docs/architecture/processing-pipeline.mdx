## End-to-End Flow

The processing pipeline handles everything from file upload to AI-generated output. Large files are automatically split for parallel processing.

## Audio Splitting

Files larger than 25MB are split by FFmpeg into chunks of up to 10 minutes or 25MB (whichever is smaller).

**Implementation**: `apps/api/src/utils/audio-splitter.ts`

```
Original file (180MB, 45 minutes)
    ├── chunk_001.m4a (0:00 - 10:00)
    ├── chunk_002.m4a (10:00 - 20:00)
    ├── chunk_003.m4a (20:00 - 30:00)
    ├── chunk_004.m4a (30:00 - 40:00)
    └── chunk_005.m4a (40:00 - 45:00)
```

Chunks are processed in parallel (configurable concurrency via `TRANSCRIPTION_CONCURRENCY`), then merged in chronological order to maintain accuracy.

## AI Analysis (GPT-5)

After transcription, the summary is always generated using GPT-5. Additional AI Assets use quality-based model selection:

| Condition | Model |
|-----------|-------|
| Primary summary | Always `gpt-5` |
| `QUALITY_MODE=premium` | `gpt-5` |
| Transcript > 10K chars | `gpt-5` |
| `QUALITY_MODE=balanced` + short transcript | `gpt-5-mini` |

**Prompt definitions**: `apps/api/src/transcription/prompts.ts`
**Analysis templates**: `apps/api/src/transcription/analysis-templates.ts`

## Error Recovery

The processor includes multiple recovery mechanisms:

1. **Job retries**: Bull automatically retries failed jobs with exponential backoff
2. **Zombie cleanup**: Hourly cron job marks stuck jobs (>24h) as FAILED
3. **Temp file cleanup**: Failed jobs trigger cleanup of intermediate chunk files
4. **Orphan detection**: Identifies and removes files in Storage without matching Firestore records
