## System Overview

Neural Summary is a monorepo application built with **Turborepo**. The system follows a queue-based architecture for reliable audio processing with real-time progress updates via WebSockets.

## Key Architectural Decisions

### Queue-Based Processing

All audio processing goes through a **Bull/Redis queue** rather than being handled synchronously. This provides:

- **Reliability**: Jobs survive server restarts and are retried on failure
- **Scalability**: Configurable concurrency (default: 2 simultaneous jobs)
- **Visibility**: Job progress is tracked and broadcast in real-time
- **Backpressure**: The queue prevents the system from being overwhelmed

### Monorepo with Shared Types

The `packages/shared` package contains all TypeScript types, constants, and validation utilities used by both frontend and backend. This ensures type safety across the full stack.

```
packages/shared/
├── src/
│   ├── types/          # Transcription, Analysis, User types
│   ├── constants/      # Subscription tiers, file limits
│   └── validation/     # Shared validation logic
├── dist/               # Built output
└── package.json
```

### Firebase as Data Layer

Firebase provides three integrated services:
- **Firestore**: NoSQL document store for all application data
- **Firebase Storage**: Blob storage for audio files with signed URLs
- **Firebase Auth**: Authentication with email/password and Google OAuth

All three share the same project configuration and SDK, simplifying the infrastructure.

| Module | Responsibility |
|--------|---------------|
| **Transcription** | Upload, processing, analysis generation |
| **Auth** | Firebase Auth token verification |
| **WebSocket** | Real-time progress broadcasting |
| **Firebase** | Firestore and Storage operations |
| **Usage** | Quota tracking and enforcement |
| **Stripe** | Subscription billing and webhooks |
| **Email** | Transactional email via Gmail SMTP |
| **Contact** | Contact form with Turnstile protection |
